1. Introduction

The Gym Management System (GymOS) is a full-stack web application built using the MERN stack (MongoDB, Express.js, React.js, Node.js). It provides a comprehensive platform for gym owners to manage members, track attendance, process payments, and assign personalized workout and diet plans. Athletes get a dedicated portal to view their schedules, log attendance, and monitor their fitness journey.

1.1 Objectives

  • Provide role-based access for Admin and Athlete users
  • Enable CRUD operations for member management
  • Implement workout and diet plan assignment with weekly scheduling
  • Track attendance with calendar-based visualization
  • Simulate payment processing with a mock payment gateway
  • Deliver a modern, responsive user interface

1.2 Scope

The system covers member registration, membership management (basic/standard/premium tiers), workout scheduling, diet planning with macro tracking, attendance logging with absence reasons, and simulated payment processing with transaction history.

2. System Architecture

2.1 Architecture Overview

The application follows a client-server architecture with a React SPA frontend communicating with a RESTful Express.js backend API, backed by a MongoDB database.

5. Frontend Design

5.1 Authentication & Routing

The app uses React Context API (AuthContext) for state management. JWT tokens are stored in localStorage and auto-attached via Axios interceptors. Protected routes enforce role-based access (admin/athlete) with automatic redirects.

6. Security Implementation

  • Password Hashing: bcryptjs with salt rounds of 10 via Mongoose pre-save hook
  • JWT Authentication: Tokens generated with 7-day expiry, verified on every protected route
  • Role-Based Access: Three middleware guards — protect (any auth user), adminOnly, athleteOnly
  • Input Validation: express-validator on auth routes for email, password, and name validation
  • CORS: Restricted to http://localhost:3000 with credentials support
  • Global Error Handler: Centralized Express error middleware returning structured JSON errors
  • Auto-logout: Axios response interceptor clears tokens and redirects on 401 responses

7. UI/UX Design

  • Design System: CSS custom properties (variables) for consistent theming across 25+ tokens
  • Typography: Inter font family with Bebas Neue for display headings
  • Neo-brutalist Style: Bold 2px borders, solid box-shadows, high-contrast color palette
  • Components: Cards with hover lift effects, color-coded stat cards, status badges, calendar grid
  • Responsive: Media queries for mobile (sidebar hidden, single-column layouts below 768px)
  • Animations: CSS keyframe animations for spinner, fade-in, slide-up, and slide-in transitions

9. How to Run

9.1 Prerequisites

  • Node.js v18+ and npm
  • MongoDB running locally on port 27017

9.2 Setup Steps

  1. Backend: cd backend && npm install && npm run seed && npm start (runs on port 5000)
  2. Frontend: cd frontend && npm install && npm start (runs on port 3000)
